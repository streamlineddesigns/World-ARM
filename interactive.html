Core Components

Each module plays a precise role in the memory-augmented prediction pipeline.

VAE
Variational Autoencoder

Compresses raw input (game frames, observations) into a structured latent space. The probabilistic encoding enables meaningful projections into keys, queries, and values — the three pillars of memory-augmented attention.

QE
Query Encoder

Transforms latent embedding + current actions into query Q. The actions are critical: they encode not just "where am I?" but "what am I doing?" — giving the memory search full situational context.

VE
Value Encoder

Produces [V] — the current state's perceptual content representation. This is what the system "sees right now," independent of memory. It gets fused with retrieved memory [M] for the final output.

SH
Spatial Hash (Memory Store)

Non-parametric episodic memory with O(1) approximate lookup. Keys (latent embeddings) are inserted MFEC-style — directly, with no gradient updates. A second hash can serve as short-term memory for few-shot adaptation.

KNN
K-Nearest Neighbors

Retrieves the K closest stored memories to the query. The spatial hash accelerates this to near-constant time. Returns candidate keys and their associated values for kernel-weighted blending.

Kw
Kernel Weighting

Applies a similarity kernel (inverse-distance, Gaussian) to weight neighbors. Closer memories dominate: the output Σ(values × weights) is a soft, experience-grounded blend rather than a single point estimate.

[M]
Memory Output

The aggregated recall vector. This is the core innovation: instead of generating predictions from learned weights alone, the system starts from what it actually remembers about similar situations.

⊕→
V ⊕ M → Output

Fusion of current perception [V] with recalled memory [M]. Can be concatenation, addition, or learned gating. The result: a prediction grounded in real experience — next state, action values, or world model forecast.

Key Concepts

The architectural decisions and principles behind World-ARM.

Output = [ V(statet) ⊕ M(Q(statet, actiont) → SH → KNN → Kernel) ] → statet+1
Core Principle

Memory-First Predictions

Traditional world models (Ha & Schmidhuber, 2018) generate future states purely from learned weights — effectively hallucinating. World-ARM starts every prediction from a retrieved memory: the spatial hash stores past (state, action, next-state) transitions. When predicting, the system first recalls similar experiences, then refines. This grounds predictions in actual experience and dramatically reduces hallucination, especially in multi-step rollouts.

Architecture

Dual-Block Design

Two communicating ARM blocks: the Action ARM for action prediction and the World ARM for future latent state prediction. This separation provides modularity, interpretability, and algorithmic control. The Action ARM decides what to do; the World ARM predicts what will happen — each grounded in their own memory stores. A residual value-conditioned network refines World ARM predictions beyond raw memory recall.

Memory Management

MFEC-Style Insertion + Dual Spatial Hash

Memory insertion follows Model-Free Episodic Control: experiences stored directly with latent keys, no gradient updates. A second spatial hash serves as short-term memory, with dynamic routing between long-term and short-term stores. When short-term wins retrieval, the system is learning from live experience — enabling few-shot adaptation to novel situations.

Future Integration

MCTS Over Memory-Grounded Predictions

With the dual-block setup, Monte Carlo Tree Search becomes natural. World ARM predicts future latent states (starting from memory, refined by residual value-conditioned projection), while Action ARM evaluates candidate moves. MCTS plans over these memory-grounded predictions — search that's anchored in experience, not hallucination. This is the natural "version 5" evolution of the architecture.

Why It Matters

The fundamental difference between predicting from weights vs. predicting from memories.

Traditional World Models

  • Predictions generated entirely from learned weights
  • Hallucination compounds over multi-step rollouts
  • No episodic grounding in actual experience
  • Slow adaptation to novel situations
  • Opaque prediction provenance — can't inspect why
  • Single monolithic prediction path

World-ARM

  • Predictions start from retrieved memories
  • Memory anchoring reduces drift over rollouts
  • Episodic store grounds predictions in real experience
  • Dual hash enables few-shot live adaptation
  • Retrievals are inspectable and interpretable
  • Modular dual-block design (Action ARM + World ARM)
